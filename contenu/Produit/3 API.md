### API

- Titre : API
- Sous-titre : Préparer l'exploitation des données de pilotage dans vos outils internes.
- Statut : En développement
- Accroche : L'API zzzbre permettra aux équipes data, développeurs et partenaires d'exploiter les données de pilotage dans leurs dashboards, analyses et automatisations.

### Exploiter vos données de pilotage

- Texte : Les usages zzzbre génèrent progressivement des données utiles pour comprendre l'entreprise : retours collaborateurs, motivation, bien-être, leadership, ateliers, priorités, décisions et plans d'action. L'API doit permettre d'exploiter ces données dans vos outils internes, sans les sortir du cadre d'accès défini dans la plateforme.

- liste :
  - Visualiser les données de pilotage dans un dashboard interne.
  - Croiser engagement, leadership, priorités et exécution.
  - Alimenter des analyses BI ou des rapports de direction.
  - Connecter zzzbre à des outils métier déjà utilisés par l'entreprise.
  - Préparer des automatisations autour du reporting ou de l'export.

### En développement

- cards :
  - Connexion avec tokens
    - Préparer un accès API contrôlé par tokens.
  - Accès aux données de pilotage
    - Récupérer les données accessibles selon les droits, rôles et périmètres configurés.
  - Export analytique
    - Faciliter l'exploitation des données dans des outils de reporting, BI ou analyse interne.
  - Automatisations futures
    - Connecter progressivement zzzbre à des workflows internes ou outils partenaires.

### Sécurité des accès API

- Texte : L'accès API devra rester cohérent avec la gestion des droits de la plateforme. L'objectif n'est pas d'ouvrir toutes les données sans contrôle, mais de permettre une exploitation encadrée selon les rôles et périmètres définis dans zzzbre.

- liste :
  - Accès par tokens.
  - Permissions liées aux droits configurés dans la plateforme.
  - Données accessibles selon les rôles, pages, secteurs ou périmètres autorisés.
  - Cohérence avec les règles d'accès décrites dans la page Sécurité.

### Exemples d'usage

- liste :
  - Alimenter un dashboard interne de pilotage.
  - Préparer un reporting BI à partir des données zzzbre.
  - Exporter des données analytiques pour un comité de pilotage.
  - Automatiser un flux entre zzzbre et un outil métier.
  - Synchroniser certaines données avec un environnement data existant.
