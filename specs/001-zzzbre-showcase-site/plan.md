# Implementation Plan: Site vitrine Zzzbre

**Branch**: `main` | **Date**: 2026-07-01 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `/specs/001-zzzbre-showcase-site/spec.md`

## Summary

Build a real public showcase site for Zzzbre with Astro.js by transposing the
existing `edito/` tree into pages, layouts, reusable components, dynamic routes,
and a typed `articles` content collection whose first entries are the Motivation
articles. The implementation must not invent public copy: every visible text,
link, status, price, contact, card, and CTA must be traceable to
`edito/structure.md`, `edito/contenu/`, or `edito/layout/`.

## Technical Context

**Language/Version**: TypeScript with Astro 7.0.4; Node.js >=22.12.0

**Primary Dependencies**: Astro core only; use `astro:content`, `astro/loaders`,
and `astro/zod` for content collections and schemas

**Storage**: Filesystem Markdown source in `edito/`; generated/normalized site
data in `src/data/`; resource articles exposed through a reusable Astro content
collection named `articles`

**Testing**: `npm run build`, source-to-site audit checklist, manual navigation
validation, mobile viewport review, CTA/contact link checks

**Target Platform**: Static public website generated by Astro

**Project Type**: Astro web application / public showcase site

**Performance Goals**: Static pages render instantly for visitors; navigation to
any top-level section is available in three interactions or fewer; no unnecessary
client-side JavaScript for content-only pages

**Constraints**: Strict source fidelity to `edito/`; no generated marketing copy;
preserve beta/development statuses; conditionally omit empty lists/blocks instead
of filling them; keep implementation Astro-first and dependency-light

**Scale/Scope**: About 33 editorial Markdown source files, 8 Motivation article
entries, one global navigation system, reusable cards/lists/CTA components, and
dynamic routes for content pages and articles

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

  files under `edito/contenu/`, and `edito/layout/1 Header.md`,
  `edito/layout/2 Footer.md`, `edito/layout/3 Sidebar.md`. The plan requires

  No additional framework dependency is planned.
- **Independent story traceability**: PASS. US1 maps to home/global navigation

  audit, navigation audit, CTA/contact link audit, mobile viewport review, and
  quickstart scenarios.
- **Security/privacy impact**: PASS. No secrets, private customer data, or
  unvalidated case studies may be introduced. Legal and security pages remain

**Structure Decision**: Use the existing single Astro application. Keep `edito/`
as the editorial source; expose normalized data in `src/data/` for concise
template rendering and conditionals; define `src/content.config.ts` with a
generic `articles` collection initially populated from the Motivation article
folder; generate generic content pages through a dynamic catch-all route while
keeping the home page explicit.
